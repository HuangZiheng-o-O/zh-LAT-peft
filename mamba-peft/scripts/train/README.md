 **各个脚本的作用**



1.  `gla_rounds.sh`  

**作用**: 这是一个核心的、分阶段的实验运行脚本。它可以通过传递一个“轮次”（Round）编号（如 bash gla_rounds.sh 1）来执行特定的一组实验。

**特点**:

 **分轮次 (Rounds)**: 它定义了多轮实验（Round 1, 2, 3, 4），每一轮包含一组不同的PEFT（参数高效微调）配置。例如，第一轮测试基础的LoRA目标模块，第二轮

测试不同的种子和配置，第三轮探索更细致的超参数（如层选择、alpha值）。

 **动态生成配置**: 如果实验所需的YAML配置文件不存在，它会根据预设的PEFT JSON模板自动创建。

 **任务可变**: 可以通过设置TASK环境变量（如 TASK=rte bash gla_rounds.sh 2）来为不同的GLUE任务（如cola, rte, mrpc等）运行实验。

**总结**: 这是主要的、结构化的实验管理脚本，用于迭代和推进研究。



2. `gla_glue_all.sh`

**作用**: 这是一个全面的、一次性跑完所有主要实验的基准测试脚本。

**特点**:

 **全面性**: 它会遍历所有的GLUE任务 (cola, rte, mrpc, sst2, qnli, qqp, mnli)。

 **固定分组**: 对每个任务，它会运行一个固定的实验分组（E0到E8），涵盖了从零样本（zero-shot）到不同LoRA模块组合的多种情况。

 **自动化**: 自动为所有任务和所有分组生成配置文件，并利用所有可用的GPU并行执行。

**总结**: 用于进行一次完整的、覆盖所有任务和核心实验设置的基准評測。



3. `gla_glue_grid.sh`

**作用**: 这是一个用于小范围“网格搜索”或特定对比实验的脚本。

**特点**: 它运行一个硬编码（predefined）的、较小的实验列表，只包含了cola和rte任务的几个特定配置。

**总结**: 不是全面的测试，而是用于快速验证或比较某些特定想法的脚本。



4. `gla_glue_ranks.sh`

**作用**: 这是一个专门用于“LoRA秩（rank）扫描”的脚本。

**特点**: 它针对cola任务，比较不同LoRA秩（r=4, r=8, r=16）在相同LoRA目标模块（qkvo）下的表现。

**总结**: 用于进行超参数（LoRA rank）调优的特定消融实验。



5. `gla_glue_targets.sh`

**作用**: 这是一个专门用于“LoRA目标模块消融”的脚本。

**特点**: 它针对cola任务，探索将LoRA应用于注意力机制的不同部分（如仅q，仅v，或qv组合等）所带来的影响。

**总结**: 用于研究LoRA作用于不同模型组件效果的特定消融实验。
